# 開発ワークフローまとめ

##  開発フローの全体像
本アプリの開発では **ロジック** と **UI** を役割分担して進める。

---

## 1. ロジック修正
- **担当**: Gemini CLI  
- **対象**:  
  - モード分岐処理  
  - progress の保存処理  
  - 複数ファイル横断修正  
- **ポイント**:  
  - 差分パッチ（patch.diff）を生成して適用  
  - logic.md を参照して仕様通りに実装  

---

## 2. UI改善
- **担当**: Gemini Code Assist  
- **対象**:  
  - ボタンレイアウト変更  
  - CSS調整（コントラスト・レスポンシブ対応）  
  - アクセシビリティ強化  
- **ポイント**:  
  - 「JSロジックは触らない」ことを指示で明記  
  - 見た目・操作性の改善に専念させる  

---

## 3. 運用ルール
1. **UIのみ修正 → Code Assist に任せる**  
   - 必ず「JSロジックは触らない」と明記すること  
2. **ロジック修正 → CLI に任せる**  
   - 差分ファイルとして出力してもらう  
3. 大きな改修や追加仕様の前に **logic.md を参照**して意図を守る  

---

## 4. 今後の応用
- Vue/React など複数ファイル構成のプロジェクトでも、CLI の横断編集が役立つ  
- Code Assist はプレビューで即見た目を確認できるので、UI改善に特化させると効率が良い
